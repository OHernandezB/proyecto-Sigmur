Arquitectura del Sistema

Ecosistema Desacoplado

Sigmur utiliza un modelo de Microservicios que permite que cada componente del sistema funcione de forma independiente. Esto facilita el mantenimiento, la escalabilidad y el despliegue continuo (CI/CD).

  • Escalabilidad Horizontal: Los servicios pueden replicarse según la demanda.
  • Resiliencia: El fallo de un servicio no detiene la plataforma completa.
  • Seguridad Perimetral: Centralizada en el API Gateway.

Front-End: Experiencia de Usuario

Potenciado por React 19 y Vite

Rendering: Client Side (CSR)

Vite & HMR

Utilizamos Vite como motor de construcción, permitiendo un Hot Module Replacement instantáneo y optimización de assets para tiempos de carga inferiores a 1s.

Tailwind & UI

Diseño atómico con Tailwind CSS. Garantiza una interfaz responsiva, moderna y consistente con un sistema de diseño corporativo legal.

Visualización GIS

Integración con Leaflet para la geolocalización de causas judiciales, permitiendo una visión territorial de la cartera de cobranza.

Back-End: El Motor del Sistema

Microservicios construidos con Java & Spring Boot

Auth Service

Gestiona la seguridad mediante JWT (JSON Web Tokens). Encargado del login, registro y validación de roles (Admin, Estudio Jurídico, Receptor).

API Gateway

Punto único de entrada. Implementa Load Balancing, Rate Limiting y enrutamiento dinámico hacia los microservicios internos.

Reporting & GIS Service

Procesamiento de datos masivos y consultas espaciales utilizando PostGIS para reportes de gestión territorial.

Workflow Service

Orquestación de procesos judiciales. Controla los estados de cada causa y las transiciones automáticas entre etapas.

Persistencia de Datos

Utilizamos Supabase Cloud como motor de persistencia relacional principal (PostgreSQL 16), extendida con PostGIS. Esta infraestructura en la nube garantiza alta disponibilidad, respaldos automáticos y una integración nativa con el sistema de autenticación.

Flujo de Datos y Operación

1. Captura y Normalización

El Frontend envía los datos de la causa (Excel o Formulario) al API Gateway. El servicio de Workflow valida la integridad de los datos.

2. Seguridad y JWT

Cada request es interceptado por un JwtFilter que valida el token emitido por Supabase Auth. Se verifica la firma y expiración antes de permitir el acceso al microservicio.

3. Persistencia Espacial

La dirección del deudor se geocodifica y se almacena como una geometría en PostGIS vinculada al registro judicial en Supabase PostgreSQL.

4. Visualización y Reporting

El FrontEnd consulta al servicio de Reporting, el cual retorna GeoJSON para renderizar los mapas de calor y KPI de gestión en tiempo real.
